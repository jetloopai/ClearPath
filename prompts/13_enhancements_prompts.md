## Prompt 10: Multi-Unit Property Support

```text
The analyzer currently assumes all properties are single-family. A massive portion of Cook County inventory is 2-4 unit buildings. Add multi-unit support.

**Your changes:**

1. Update `apps/analyzer/src/components/ConfigModal.tsx`:
   - Add a "Units" selector between the Condition selector and the Advanced Options toggle.
   - Style it as a row of 4 toggle buttons matching the condition selector design: `1 Unit`, `2 Units`, `3 Units`, `4 Units`.
   - Default to `1`. Store in state as `units`.
   - Include `units` in the POST body sent to `/api/analyze`.

2. Update `apps/analyzer/src/app/api/analyze/route.ts`:
   - Accept `units` from the request body (default to 1).
   - Adjust the calculations:
     - **Rent estimate:** Multiply by units. A 3-unit building generates 3x rent. `rentEstimate = Math.round(arv * rentRatio) * units`
     - **Rehab estimate:** For multi-unit, scale rehab by a factor. Use: `rehabEstimate = baseRehabEstimate * (1 + (units - 1) * 0.65)`. This assumes each additional unit adds ~65% of the base rehab cost (shared systems like roof, HVAC, foundation are already covered).
     - **Expenses:** Vacancy, management, maintenance, and capex all scale with the higher rent number (they're percentages of rent, so this happens automatically). Insurance should scale: `insuranceMonthly * units`. Taxes stay the same (based on ARV, not units).
     - **Cash flow and CoC:** These naturally update since they derive from rent and expenses.
     - **Flip profit:** Stays the same formula (ARV - costs). ARV for multi-unit is already based on purchase price × multiplier which the user sets. The rehab increase is captured above.
   - Include `units` in the `inputs` JSONB field saved to Supabase.

3. Update `apps/analyzer/src/components/ResultsView.tsx`:
   - In the property summary line ("3 Bed • 2 Bath • 1,400 SqFt • Built 1965"), prepend the unit count if > 1: e.g., "2-Unit • 3 Bed • 2 Bath • 1,400 SqFt • Built 1965".
   - The rental card should show per-unit rent as well: "Rent: $1,566/unit/mo ($3,132 total)".

4. Update the `StoredAnalysis` interface and sessionStorage payload to include `units`.

5. Update `apps/analyzer/src/lib/calculations.ts` type `AnalysisResults` if needed to include the unit count.

Test with a 2-unit and a 4-unit property. Verify the rent and rehab scale correctly and the results page reflects the multi-unit data.
```
